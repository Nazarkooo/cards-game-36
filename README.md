# Брідж — онлайн

Карткова гра на колоду 36 карт (гібрид Дурака/Уно з власними правилами: 6/7/8/A/J-ефекти, Бридж, бонус валета, накопичувальний рахунок) для 2–5 гравців, кожен на своєму пристрої.

## Структура

- `server/` — Node.js + TypeScript, Express + Socket.IO. Авторитетна ігрова логіка (`src/gameEngine.ts`), стан кімнат у пам'яті.
- `client/` — React + TypeScript + Vite, Socket.IO-client. UI адаптивний (мобільний і десктоп).

## Запуск локально

У двох терміналах:

```bash
# термінал 1 — сервер (порт 3001)
cd server
npm install
npm run dev

# термінал 2 — клієнт (порт 5173, проксує /socket.io на 3001)
cd client
npm install
npm run dev
```

Відкрийте `http://localhost:5173` у кількох вкладках/браузерах, щоб перевірити гру кількома гравцями.

### Перевірка правил рушія без UI

```bash
cd server
npm run test:engine
```

Запускає `src/engineTest.ts` — ручні перевірки ланцюжка 7, пропуску ходів через A, детекції "Бридж", бонусу валета, порогів рахунку 295/300.

## Деплой (один сервіс, наприклад Render.com)

Сервер після білда клієнта віддає статичні файли (`client/dist`) і той самий порт обробляє Socket.IO — без CORS, один URL.

1. Запуште проєкт у Git-репозиторій (GitHub/GitLab).
2. На [Render.com](https://render.com) створіть **Web Service** з цього репо.
3. Налаштування:
   - **Build Command**: `cd client && npm install && npm run build && cd ../server && npm install && npm run build`
   - **Start Command**: `cd server && npm start`
   - **Environment**: Node
   - Render автоматично виставляє `PORT` — сервер його використовує (`server/src/server.ts`).
4. Після деплою отримаєте один URL — поділіться ним з гравцями, кожен заходить зі свого пристрою.

> Важливо: потрібен хостинг із постійним з'єднанням (Render, Railway, Fly.io). Serverless-платформи типу Vercel/Netlify Functions не підтримують WebSocket-з'єднання Socket.IO в довготривалому режимі.

## Правила (повний опис)

### Базовий хід
На верхню карту столу (ранг R, масть S) можна класти: карту тієї ж масті S, карту того ж рангу R, або Валета (завжди можна, оголошує нову активну масть). Це базове правило діє для будь-якого рангу, якщо нижче не сказано інше.

### Якщо нема чим ходити
На звичайному ході (не під обов'язком 6) можна добровільно взяти карту з колоди навіть маючи хід — це вибір гравця. За хід можна взяти **лише 1 карту**: після добору або граєш (якщо тепер є хід), або, якщо й після добору нічим ходити, хід переходить далі. Виняток — **перший хід нової роздачі**: якщо стартеру (гравцю з 4 картами) нічим докинути, він може просто пропустити хід без жодного добору.

### Спецефекти карт
- **6** — хто кинув 6 (або кому дісталась 6 як стартова карта роздачі), той сам зобов'язаний негайно продовжувати накривати (тою ж мастю, ще однією 6, або Валетом), тягнучи з колоди без обмежень, поки не накриє. Лише після цього хід переходить далі.
- **7** — можна кинути кілька 7 одним ходом. Той, кому дісталась 7 (чи кілька), або тягне 2 карти за кожну кинуту 7 і пропускає хід, або переводить далі своїми 7 (сума тягнутих карт накопичується по всьому ланцюжку, поки хтось не візьме замість переведення).
- **8** — можна кинути кілька 8 одним ходом. Наступний гравець тягне по 1 карті за кожну кинуту 8, але хід **не** пропускає: одразу пробує зіграти; якщо нічим — тягне ще 1 карту (обов'язково); якщо й тоді нічим — хід переходить далі.
- **A** — можна кинути кілька A одним ходом: стільки наступних гравців пропускають хід, скільки A кинуто.
- **J (Валет)** — покриває будь-яку карту, оголошує нову активну масть; можна кинути кілька Валетів одним ходом, фінальна масть — з останнього.

### Стартова карта роздачі діє "як щойно зіграна"
Карта, яку роздача кладе на стіл (5-та карта гравця-стартера), несе свій спецефект автоматично, навіть якщо стартер зіграє щось інше:
- **6** — стартер сам зобов'язаний накрити (як і завжди для 6).
- **7** — наступний гравець автоматично стає перед вибором "тягни 2 чи переводь"; якщо стартер додатково докидає свою 7 — сума накопичується (наприклад: стартова 7 + ще одна докинута = 4 карти).
- **8** — наступний гравець автоматично тягне 1 картку; якщо стартер докидає свою 8 — тягне вже 2 (стартова + докинута).
- **A** — наступний гравець автоматично пропускає хід; докинута стартером своя A додає ще один пропуск.
- **J** — стартер зобов'язаний спочатку оголосити масть (бо в роздачі вона ще не визначена) — або просто оголосити, або кинути свого Валета і одразу оголосити. Без цього інші дії неможливі.

### Очки за картами
6,7,8,9 = 0; 10 = 10; J = 20 (J♠ = 40); Q = 10 (Q♠ = 60); K = 10 (всі масті); A = 15 (всі масті). На кожній картці видно бейдж з очками (порожньо для 0-очкових), число автоматично множиться на поточний множник раунду.

### Роздача карт
- **Перший раунд сесії**: один випадковий гравець отримує 4 карти (5-та лежить на столі), решта — по 5.
- **Кожен наступний раунд**: переможець попереднього раунду отримує 4 карти (5-та на столі), решта — по 5.
- Гравець із 4 картами завжди ходить **першим**.

### Кінець раунду
Раунд закінчується, коли гравець скидає останню карту (через звичайний хід; через 8 — одразу після примусового добору наступним гравцем; через 7 — лише коли хтось у ланцюжку реально візьме накопичені карти замість переведення, а не раніше).
- Переможець не отримує очок за свою руку.
- Інші гравці отримують суму номіналів карт на руках, помножену на поточний множник раунду.
- **Якщо переможець закінчив одним або кількома Валетами** — він обирає: "усім +X" або "собі -X", де X = сума значень усіх викинутих Валетів (20 за звичайний, 40 за піковий; наприклад хресний+піковий = 20+40=60), помножена на поточний множник раунду.

### Бридж
Якщо останні 4 карти в стосі на столі одного рангу — будь-який гравець може натиснути кнопку "Бридж". Раунд миттєво завершується: очки всіх гравців (включно з тим, хто оголосив) рахуються з карт на руках, помножені на (поточний множник раунду × 2).

### Множник раунду
Коли колода добору закінчується, стіс (крім верхньої активної карти) перемішується в нову колоду добору, множник раунду зростає (×2, ×3, ×4...) і застосовується до всього підрахунку очок до кінця поточного раунду. Новий раунд завжди починається з ×1.

### Накопичувальний рахунок і сесія
Після кожного раунду: якщо рахунок гравця стає рівно 295 — обнулюється до 0; якщо більше 300 — гравець вилітає з сесії. Сесія триває кілька роздач, доки не залишиться 1 гравець (переможець сесії).

### Інше
- **Чат** у кімнаті (доступний у лобі й під час гри).
- **Перепідключення**: гравець ідентифікується стійким id у localStorage, тож релоад сторінки під час гри відновлює його місце з тою ж рукою (кімната живе ще 30с після дизконекту).
