# Карти 36 — онлайн

Карткова гра на колоду 36 карт (гібрид Дурака/Уно з власними правилами: 6/7/8/A/J-ефекти, Бридж, бонус валета, накопичувальний рахунок) для 2–5 гравців, кожен на своєму пристрої.

## Структура

- `server/` — Node.js + TypeScript, Express + Socket.IO. Авторитетна ігрова логіка (`src/gameEngine.ts`), стан кімнат у пам'яті.
- `client/` — React + TypeScript + Vite, Socket.IO-client. UI адаптивний (мобільний і десктоп).

## Запуск локально

У двох терміналах:

```bash
# термінал 1 — сервер (порт 3001)
cd server
npm install
npm run dev

# термінал 2 — клієнт (порт 5173, проксує /socket.io на 3001)
cd client
npm install
npm run dev
```

Відкрийте `http://localhost:5173` у кількох вкладках/браузерах, щоб перевірити гру кількома гравцями.

### Перевірка правил рушія без UI

```bash
cd server
npm run test:engine
```

Запускає `src/engineTest.ts` — ручні перевірки ланцюжка 7, пропуску ходів через A, детекції "Бридж", бонусу валета, порогів рахунку 295/300.

## Деплой (один сервіс, наприклад Render.com)

Сервер після білда клієнта віддає статичні файли (`client/dist`) і той самий порт обробляє Socket.IO — без CORS, один URL.

1. Запуште проєкт у Git-репозиторій (GitHub/GitLab).
2. На [Render.com](https://render.com) створіть **Web Service** з цього репо.
3. Налаштування:
   - **Build Command**: `cd client && npm install && npm run build && cd ../server && npm install && npm run build`
   - **Start Command**: `cd server && npm start`
   - **Environment**: Node
   - Render автоматично виставляє `PORT` — сервер його використовує (`server/src/server.ts`).
4. Після деплою отримаєте один URL — поділіться ним з гравцями, кожен заходить зі свого пристрою.

> Важливо: потрібен хостинг із постійним з'єднанням (Render, Railway, Fly.io). Serverless-платформи типу Vercel/Netlify Functions не підтримують WebSocket-з'єднання Socket.IO в довготривалому режимі.

## Правила (короткий підсумок)

- Базовий хід: на верхню карту можна класти карту тієї ж масті, тієї ж рангу, або Валета (вайлд, оголошує нову масть).
- **6** — обов'язково накрити (масть/6/валет), інакше тягнеш карти з колоди, доки не накриєш.
- **7** — наступний гравець тягне 2 карти за кожну кинуту 7 і пропускає хід, або переводить своїми 7.
- **8** — наступний гравець тягне 1 карту за кожну кинуту 8, але хід не пропускає.
- **A** — кидок з N асів пропускає хід N наступним гравцям.
- **J** — вайлд-карта, оголошує нову активну масть.
- Очки: 6-9 = 0, 10 = 10, J = 20 (J♠ = 40), Q = 10 (Q♠ = 60), K = 10, A = 15.
- Роздача нового раунду: переможець попереднього раунду отримує 4 карти (5-та лежить на столі), решта — по 5.
- Кінець раунду: переможець не отримує очок за свою руку; інші — суму номіналів карт на руках. Якщо переможець завершив Валетом — обирає бонус (+20/+40 усім або -20/-40 собі).
- **Бридж**: якщо останні 4 карти на столі одного рангу — будь-хто може оголосити Бридж, раунд завершується миттєво, очки всіх гравців (разом з тим, хто оголосив) рахуються з усіх карт на руках і множаться на ×2.
- Колода добору закінчилась → стіс перемішується, множник раунду зростає (×2, ×3...), застосовується до фінального підрахунку.
- Рахунок > 300 → вильот із сесії. Рахунок = 295 → обнулення до 0. Сесія триває, доки не залишиться 1 гравець.

## Відомі обмеження MVP

- Немає перепідключення після оновлення сторінки (id гравця = id socket-з'єднання) — якщо гравець оновить сторінку під час гри, він зайде як новий гравець.
- Немає збереження сесій між перезапусками сервера (стан кімнат у пам'яті).
